무엇을 도와드릴까요?

발송, 인증, 스팸함 도달률 등 메일 서비스 이용 중 자주 묻는 질문을 모았습니다. 원하는 답변을 찾지 못하셨다면 아래에서 문의해 주세요.

발신 인증 (SPF/DKIM)

SPF란 무엇인가요?

SPF(Sender Policy Framework)는 "이 도메인이 어떤 서버에서 메일을 보내도 되는지"를 DNS에 등록해두는 인증 방식입니다. 수신 서버는 메일이 도착하면 발신 서버의 IP가 SPF 목록에 있는지 확인합니다.

저희 서비스를 이용하시려면 도메인의 DNS에 아래와 같은 TXT 레코드를 추가해야 합니다.

v=spf1 include:mail.example.com ~all

등록 후 반영까지 최대 24~48시간이 걸릴 수 있습니다.

DKIM은 왜 설정해야 하나요?

DKIM(DomainKeys Identified Mail)은 메일 본문이 발송 이후 위·변조되지 않았음을 증명하는 전자서명입니다. SPF와 함께 설정하면 발신자 신뢰도가 크게 올라가고, 스팸함으로 분류될 확률이 낮아집니다.

대시보드 > 도메인 설정 > DKIM에서 발급된 CNAME 값을 DNS에 등록하면 자동으로 활성화됩니다.

DMARC은 꼭 설정해야 하나요?

필수는 아니지만 강력히 권장합니다. DMARC은 SPF·DKIM 인증에 실패한 메일을 수신 서버가 어떻게 처리할지(격리/거부/허용) 지정하는 정책입니다. 2024년 이후 Gmail·Yahoo 등 주요 메일 서비스는 대량 발송 시 DMARC 설정을 사실상 필수로 요구하고 있습니다.

권장 시작 값: v=DMARC1; p=none; rua=mailto:dmarc@yourdomain.com

발송 문제

메일이 발송되지 않아요.

다음 순서로 확인해 주세요.

  1. 대시보드 > 발송 로그에서 상태가 Failed인지 Queued에 머물러 있는지 확인
  2. 도메인 인증(SPF/DKIM) 상태가 "활성"인지 확인
  3. API를 사용 중이라면 응답 코드가 4xx/5xx인지 확인 (아래 오류 코드 문서 참고)
  4. 일일/월간 발송 한도를 초과하지 않았는지 확인

위 항목을 모두 확인했는데도 해결되지 않으면 발송 로그의 메시지 ID와 함께 문의해 주세요.

발송은 성공했는데 상대방이 못 받았대요.

발송 성공은 저희 서버가 수신 서버에 메일을 정상적으로 전달했다는 의미이며, 이후 처리는 수신 측 정책에 따라 달라집니다. 가장 흔한 원인은 스팸함 분류이며, 아래 '스팸함·도달률' 항목을 참고해 주세요. 수신자에게 스팸함도 함께 확인해달라고 요청해 보세요.

메일이 반송(bounce)되었어요. 원인이 뭔가요?

반송에는 두 가지 유형이 있습니다.

  • Hard bounce: 존재하지 않는 주소, 도메인 오류 등 영구적 실패 (자동으로 발송 목록에서 제외됩니다)
  • Soft bounce: 수신함 용량 초과, 일시적 서버 장애 등 (일정 시간 후 자동 재시도)

구체적인 반송 사유는 대시보드 > 발송 로그 > 상세보기에서 SMTP 응답 코드로 확인할 수 있습니다.

스팸함·도달률

메일이 자꾸 스팸함으로 들어가요.

주요 원인은 다음과 같습니다.

  • SPF/DKIM/DMARC 미설정 또는 설정 오류
  • 새 도메인·IP로 대량 발송 (워밍업 필요)
  • 수신자의 낮은 참여율(열람·클릭 없음) 누적
  • 스팸성 단어, 과도한 느낌표·이미지 비율
  • 수신 거부한 주소로 반복 발송

신규 발신 도메인은 첫 2~4주간 소량으로 시작해 점진적으로 발송량을 늘리는 IP 워밍업을 권장합니다.

발신 평판 점수는 어디서 확인하나요?

대시보드 > 도메인 평판 메뉴에서 열람률, 반송률, 스팸 신고율, 발신 IP 상태를 종합한 점수를 확인할 수 있습니다. 스팸 신고율이 0.1%를 넘으면 경고가 표시되니 발송 리스트 품질 관리가 중요합니다.

계정·API

API 키는 어디서 발급받나요?

대시보드 > 설정 > API 키에서 발급할 수 있습니다. 키는 발급 시 한 번만 노출되므로 안전한 곳에 보관해 주세요. 노출이 의심되면 즉시 재발급(기존 키 폐기)하시는 것을 권장합니다.

API 요청 시 401/403 오류가 발생해요.

401은 API 키가 누락되었거나 잘못된 경우, 403은 키는 유효하지만 해당 리소스(예: 미인증 도메인)에 대한 권한이 없는 경우입니다. 요청 헤더에 Authorization: Bearer YOUR_API_KEY가 정확히 포함되어 있는지 확인해 주세요.

여러 도메인을 하나의 계정에서 관리할 수 있나요?

네, 가능합니다. 대시보드 > 도메인 관리에서 도메인을 추가하고 각각 별도로 SPF/DKIM 인증을 진행하시면 됩니다. 요금제에 따라 등록 가능한 도메인 수가 다를 수 있습니다.

요금·발송량

일일/월간 발송 한도는 어떻게 되나요?

요금제에 따라 한도가 다르며, 현재 사용량은 대시보드 상단에서 실시간으로 확인할 수 있습니다. 한도 초과가 예상되시면 요금제 업그레이드 또는 한도 상향을 미리 요청해 주세요.

무료 플랜에서 유료 플랜으로 전환하면 발송 중인 메일에 영향이 있나요?

플랜 전환은 즉시 적용되며 진행 중인 발송 작업에는 영향을 주지 않습니다. 결제는 전환 시점부터 일할 계산되어 청구됩니다.

검색 결과가 없습니다. 다른 키워드로 시도해 보세요.
